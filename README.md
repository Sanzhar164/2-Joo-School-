# JOO School — дашборд отзывов 2ГИС

Три файла:
- `joo-dashboard.html` — сам дашборд
- `collect.py` — сборщик отзывов с 2ГИС → `data.json`
- `.github/workflows/collect.yml` — автозапуск сборщика на GitHub (бесплатно)

Дашборд при открытии пытается подгрузить `data.json`. Если файла нет —
показывает встроенные данные (зашитые на 28 мая 2026). В шапке видно,
какие данные используются и когда обновлены.

---

## Вариант A. Разово / вручную обновить данные (на своём компьютере)

1. Установи Python 3 и зависимость:
   ```
   pip install requests
   ```
2. Собери свежие данные:
   ```
   python collect.py
   ```
   Появится `data.json`.
3. Запусти локальный веб-сервер в этой папке (нужно, чтобы браузер
   мог прочитать `data.json` — двойной клик по html не сработает из-за
   ограничений браузера):
   ```
   python -m http.server 8000
   ```
4. Открой в браузере: http://localhost:8000/joo-dashboard.html

Чтобы обновить — просто снова `python collect.py` и обнови страницу.

---

## Вариант B. Автомат и бесплатно (GitHub Pages + Actions)

Так дашборд будет жить по ссылке и обновляться сам раз в день.

1. Создай репозиторий на GitHub, залей туда все 3 файла
   (`joo-dashboard.html`, `collect.py`, папку `.github/workflows/`).
2. Settings → Pages → Source: ветка `main`, папка `/root`. Сохрани.
   Получишь ссылку вида `https://ТВОЙ_ЛОГИН.github.io/РЕПО/joo-dashboard.html`.
3. Settings → Actions → General → Workflow permissions →
   включи **Read and write permissions** (чтобы бот мог коммитить data.json).
4. Вкладка Actions → выбери «Сбор отзывов 2ГИС» → Run workflow (первый запуск вручную).
   Дальше он будет сам запускаться каждый день в ~09:00 по Алматы.

Готово: открываешь ссылку — всегда свежие отзывы.
Частоту меняешь в `collect.yml` (строка `cron`).

---

## Важные оговорки

- API отзывов 2ГИС публичный, но неофициальный: 2ГИС может ограничить
  или изменить доступ. Для 8 точек раз в день — обычно без проблем.
- Тянется до 100 последних отзывов на филиал (лимит постранички API).
- «Число оценок» (votes) API в открытом виде не отдаёт — оно зашито
  как последнее известное значение; рейтинг и отзывы обновляются с API.
- Сентимент и темы считаются по ключевым словам (рус+каз) — это оценка,
  не точный ML-анализ.
